백틱

  • ES6에 새로운 종류의 문자열 표기법을 도입

    • ‘(작은 따옴표)와 “(큰 따옴표)가 아닌 `(백틱) 문자를 사용함

      • 기존과 동일하게 사용 가능

  • 활용과 사용법

    • 문자열 채워넣기(string interpolation) 기능 사용 가능!

      • 이것은 +연산자보다 약간 더 좋은 문법일 뿐이지만…
    • 템플릿 대입문에 어떤 javascript 표현이 올 수 있다.
      • 함수 호출 구문, 수식 구문 등 모든 구문이 허용됨
    • 템플릿 대입문에 오는 값이 문자열이 아니라면 그 값은 통상적인 규칙에 따라 문자열로 변환될 것이다. (.toString())
    • 템플릿 문자열 안에 `, $, { 를 써야할 필요가 있다면 백스플래쉬 문자를 이용한 이스캐이프 표현을 사용합니다.

    • 여러 줄에 걸쳐 표현 가능

      • 줄바꿈, 들여쓰기 등 템플릿 문자열 속 모든 화이트 스페이스들이 그대로 포함됨

  • 백틱의 미래

    • 템플릿 문자열이 하지 못하는 일들에 대하여 이야기해 봅시다

      • 템플릿 문자열은 특수 문자들을 자동으로 이스케이프시켜 표현해주지 않는다.
      • 크로스-사이트 스크립팅(cross-site scripting) 공격을 피하려면, 신뢰할 수 없는 데이터에 대한 처리를 직접해주어야 한다.
      • 템플릿 문자열과 다국어 처리 라이브러리 사이의 상요장용에 대해서는 분명히 정의된 것이 없다.
      • 템플릿 문자열은 언어권별 숫자 포맷과 날짜 포맷을 알아서 처리해 주지 않는다.
      • 템플릿 문자열은 루프문은 물론 조건문을 위한 문법을 가지고 있지 않다.

    • 태그된 탬플릿(tagged template)

      • 템플릿 문자열의 한계를 개선 하는 것
      • 문법
        • 백틱 앞에 태그를 하나 더 붙인 문자열
          • SaferHTML
            • ES6에서 등장한 키워드는 아님

      • SaferHTML

        • 우리는 이 태그를 이용해서 앞에서 나열한 템플릿 문자열 한계들 중 첫번째 한계를 극복할 것이다.
          • 여기에 사용된 SaferHTML 태그는 단순한 형태의 식별자이다
        • SaferHTML.escape 같은 속성형태도 태그로 사용될 수 있다.

        • SaferHTML.escape({unicodeControlCharacters: false}) 같은 메소드 호출 형태도 태그로 사용할 수 있다.

          • 위와 동일한 코드

          • templatedata

            • 템플릿 문자열 파트로 이루어진, 값을 변경할 수 없는 배열
            • templateData 배열은 2개의 요소를 가진다.
              • Object.freeze([“

                “, “ has sent you a bonk.

                “]
            • 사실 한가지 속성이 더있다.
              • raw
                • 실제 줄바꿈 문자 대신 \n로 표현된 문자열을 담고 있다. 표준 태그인 String.raw가 이 raw 문자열을 이용한다.
        • SaferHTML 태그는 함수를 통해 문자열과 대입문을 무한히 다양하게 해석하는 수단으로 활용할 수 있다.

      • i18n 태그를 이용해 다국어 지원가능

      • hashTemplate 태그로 태그 만들기

        • 루프문이나 조건문을 지원을 안할때 만들어 사용하는 방법

          • 정규식, DOM 트리, 이미지, 비동기 작업 전체를 대표하는 프라미스, JS 데이터 구초체ㅡ GL 쉐이더 등 어떤 것을 만들 때도 태그를 이용 할 수 있다.
