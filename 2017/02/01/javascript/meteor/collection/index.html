컬렉션이란?

  • 컬렉션은 데이터 구조체다
  • 컬렉션은 어떤 앱에서나 중심적인 역할을 한다.
  • 컬렉션은 데이터 입출력을 처리한다.
  • 컬렉션은 앱 전체에 사용되므로 var키워드를 사용하지 않는다.
  • 컬렉션은 여러 저장소에 저장될 수 있다.
  • 컬렉션은 양쪽 코드에서 실행된다.
  • 컬렉션은 서버에서 몽고디비와 연결되어 데이터 입출력한다.
  • 컬렉션은 클라이언트에서 서버의 데이터를 일부분 복제해서 가지고 있는 것.

WebApp의 데이터 저장

  • 브라우저 메모리 : 메모리상에 저장하는 것
  • 브라우저 저장소 : 브라우저 로컬 스토리지
  • 서버 데이터베이스 : 몽고디비
    • 컬렉션은 이 3가지에 모두 저장될 수 있다.

클라이언트-서버 통신(실습)

  • 핵심은 동일한 컬렉션간의 동기화
    • 미티어가 해준다.
  • 서버에서의 컬렉션
    • 컬렉션은 MongoDB로의 API로 기능한다.
    • 컬렉션은 MongoDB의 API다.
  • 클라이언트에서의 컬렉션
    • 컬렉션은 클라이언트에서 캐시라고 말한다.
    • 컬렉션을 캐시라고 부르는 이유는 데이터의 부분집합을 가지며 빠르게 접근할 수 있어서이다.
