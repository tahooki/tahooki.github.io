심볼 (Symbol)

  • javascript ES6 이전 타입 6가지
    • Undefined
    • Null
    • Boolean
    • Number
    • String
    • Object
  • 타입이란?
    • 어떤 값들의 집합.
  • 7번째 타입(type)
    • 심볼이란?
      • 어떤 값.
      • 심볼(Symbol)은 프로그램이 이름 충돌의 위험 없이 속성(property)의 키(key)로 쓰기 위해 생성하고 사용할 수 있는 값
    • 심볼은…
      • 생성되면 변경되지 않는다.
      • 속성을 부여할 수 없다.
      • 속성 이름으로 사용할 수 있다.
      • 문자열과 유사하지만 각각의 심볼은 고유하다.
        • 같은 주석을 가지고 있어도 다르다.
  • 왜 사용?
    • 코드 충돌을 피하기위해서
    • 예를들어 같은 다른 라이브러리 작성자가 해당 element를 접근해서 같은 동작을 취해 검사하거나 조작할경우 자신의 코드와 충돌할 수 있다.
      • 이점을 회피하기 위한 기법.
  • 물론 다른 방법도 있지만…
    • 디버깅이 나쁘다. 난수만 보여지기 때문에.
      • 그래서 등장하게 된것이 심볼.

  • 어떻게 사용?

    • 기본적인 방법

      • 심볼 값을 생성해서 받는다.
      • 이 값은 어떠한 값과도 다르다.
      • 속성의 키로 사용할 수 있다.
      • 속성이 심볼을 키로 받으면 다른 속성과 충돌되지 않을 것을 보장 받는다.

    • 고유 심볼 생성

      • 주석을 달아 놓은 심볼.
        • 디버깅할때 유용하다.
          • .toString() 함수를 써서 심볼값을 문자열로 바꿀때
          • 에러메세지를 참조할 때
          • cosnole.log()로 찍을 때
    • if (isMoving in element)처럼 속성을 참조
    • element[isMoving]처럼 속성을 제거하는 것도 가능
  • 심볼을 간단한 캡슐화(encapsulation) 매카니즘으로 사용할 수 있다.
    • 모듈이 스스로 심볼을 만드는 경우 해당 모듈은 해당 심볼을 모든 객체에 적용할 수 있다.
      • 충돌하는 것을 전혀 걱정할 필요가 없다.
        • 자동으로 어떻게…생성하게 하는걸까?

  • 주의사항

    • 문자열로 자동변환되지 않는다.

      • String(sym) 또는 sym.toString()을 써서 심볼을 명시적으로 문자열로 변환하면 TypeError를 피할 수 있다.
  • 심볼의 3가지
    • Symbol()
      • 고유심볼
    • Symbol.for(string)
      • 심볼 레지스트리에 있는 심볼 목록을 참조하여 생성한다.
      • 여러페이지에 걸쳐 같은 심볼을 생성해야 할 경우 유용하다.
    • Symbol.iterator
      • 표준에 의해 정의된 심볼을 사용한다.
        • 특별한 용도가 있다는데 잘 모르겠다.
  • ES6 스펙이 심볼을 사용하는 방식 - (무시해도 될듯하다)
    • instanceof를 확장 가능하게 만든다.
      • ES6에서, object instanceofconstructor 구문은 생성자(constructor)의 메소드인constructorSymbol.hasInstance로 규정됩니다. 이는 이 구문이 확장 가능하다는 의미입니다.
        • ??? 무슨말인지 모르겠슴
    • 새로운 기능과 예전 코드의 충돌을 제거한다.
      • 잘 알려져있지 않지만, 우리는 어떤 ES6 Array 메소드들이 단지 정의되어 있다는 이유로 기존 웹사이트들의 동작에 오류를 일으킨다는 사실을 발견했습니다. 다른 웹 표준들에도 비슷한 문제가 있었습니다. 단지 브라우저에 새로운 메소드를 추가했을 뿐인데 기존 사이트들의 동작이 방해 받는 것입니다. 동작 오류의 주요 원인은 동적 스코핑(dynamic scoping) 기능 때문이었습니다. 그래서 ES6는 특별한 심볼을 도입했습니다. Symbol.unscopables입니다. 웹 표준은 Symbol.unscopables을 이용해서 특정 메소드들이 동적 스코핑에 관여되는 것을 방지할 수 있습니다.
        • 라는데 잘알려지지 않은거라고 해서 무시할까 함.
    • 새로운 종류의 문자열 검색(string-matching)을 지원한다.
      • ES5에서str.match(myObject) 코드는 myObject를 RegExp로 변환시키려고 시도합니다. ES6에서 이 코드는 우선 myObject가 myObjectSymbol.match 메소드를 갖고 있는지 확인합니다. 이제 라이브러리들은 RegExp 객체가 사용되는 모든 곳에 커스텀 문자열 파싱(string-parsing) 클래스를 제공할 수 있습니다.
